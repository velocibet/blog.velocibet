## 왜 메신저를 만들게 됐을까

저는 어렸을 때부터 코드를 짜서 무언가를 만드는 것을 매우 좋아했습니다. 초등학교 때 Roblox라는 플랫폼에서 게임을 만드느라 그때 처음 코딩을 접하게 되었습니다.

Roblox에서 Lua라는 프로그래밍 언어를 사용해서 그 언어를 처음으로 배우고 여러가지 게임을 만들었던 기억이 나는데요. 중학교때는 학교 끝나자마자 집에 오면 코딩부터 했던 기억이 납니다. ㅋㅋㅋ

![로블록스로 만든 게임](images/post1-01.png)

그러다가 Roblox가 질릴때 쯤 웹 개발에 대해 알게 됩니다. 처음에는 HTML, CSS, JS만 알면 누구나 웹사이트를 만들수 있다는게 다른 언어에 비해 쉽고 간단해보였죠. 처음엔 'W3School' 이라는 사이트에서 HTML의 기본 태그들과 CSS 속성, JS 기본 문법 들을 공부했습니다.

그리곤 자연스럽게 프래임워크들에 대해서도 알게됐고 React, Vue같은 프래임워크를 경험해보게 되었습니다.

작년에 Vue를 이용해서 간단한 웹사이트를 만든적이 있습니다. 그때는 그냥 마음대로 꾸미고 싶은대로 꾸민 개인 사이트였는데 사람의 욕심이라 할까요? 점점 복잡한 기능을 넣기 시작하다보니 갑자기 자신감이 들기 시작했습니다. 이 과정에서 여러가지 웹개발 기술도 알게 되니까 실제 서비스도 만들수 있겠더라고요.

그래서 웹으로 가볍게 쓸 수 있는 메신저를 직접 만들어보고 싶었습니다. 이미 잘 만들어진 서비스는 많지만, **직접 설계해보는 경험**이 목표였습니다.

## 기술 스택

### 프론트엔드
- **Nuxt**

### 백엔드
- **NestJS, PostgreSQL**

### 스토리지
- **Cloudflare R2**

Nuxt는 제가 Vue를 많이 사용하기도 했고, 상태 관리, 라우터 등 개발하기 편한 기능이 기본적으로 들어가있어 고르게 됐습니다. NestJS 같은 경우는 백엔드로 Node.js를 사용할 생각인데 Express같은 경우는 기본적으로 TypeScript를 지원하지 않아 개발할 때 규모가 커지면 코드가 난해해질거 같다는 생각이 들었습니다. 따라서 SpringBoot같이 Layered Architecture를 지원하는 NestJS가 이번 프로젝트에 적합하다고 보았습니다.

DB는 처음에 MySQL를 사용했었습니다.. 그런데 제가 얼마전에 노트북 OS를 개발 편의를 위해 Arch Linux로 마이그래이션을 했는데 이게 웬걸.. Arch Linux에선 MySQL을 사용하지 못한다는 겁니다. 그래서 강제로 PostgreSQL을 사용하게 됐습니다. 근데 사용해보니까 PostgreSQL이 더 좋은거 같긴 합니다.

스토리지 같은 경우는 AWS S3같은 경우 무료 티어에 한계가 있지만 Cloudflare R2 같은 경우 무료티어에는 20GB 까지 저장공간을 주기 때문에 R2를 고르게 됐습니다.

최대한 단순하게 시작해서 필요한 기능만 하나씩 붙여나갈 생각입니다.
