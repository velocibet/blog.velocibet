## JavaScript 주석이란?

주석은 **코드를 설명하거나 메모를 남기는 방법**입니다.  
주석은 프로그램 실행 시 무시되므로, 코드의 동작에 영향을 주지 않습니다.

주석은 다음과 같은 목적으로 사용됩니다:
- 코드의 목적 설명
- 복잡한 로직 이해하기
- 나중에 수정할 부분 표시
- 협업 시 다른 개발자에게 정보 전달

---

## 주석의 종류

### 1. 한 줄 주석 (//)

한 줄만 주석 처리할 때 사용합니다.

```javascript
// 사용자의 이름을 저장합니다
let userName = "Alice";

let age = 25; // 사용자의 나이

// 다음은 사용자 정보를 출력합니다
console.log(userName);
```

슬래시 `//` 뒤의 모든 텍스트는 주석입니다.

### 2. 여러 줄 주석 (/* */)

여러 줄을 주석 처리하거나 함수 설명을 쓸 때 사용합니다.

```javascript
/*
사용자 정보를 처리하는 함수입니다.
입력받은 이름과 나이를 콘솔에 출력합니다.
*/
function printUserInfo(name, age) {
  console.log(name + "은 " + age + "살입니다.");
}

/*
다음 코드는 임시로 비활성화했습니다.
추후 기능 추가 시 사용할 예정입니다.
let tempData = 123;
let tempName = "Test";
*/
```

## 주석을 효과적으로 사용하기

### 좋은 주석의 예

```javascript
// 이메일이 유효한 형식인지 확인합니다
function isValidEmail(email) {
  const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  return emailRegex.test(email);
}

// API에서 사용자 데이터를 불러옵니다
async function fetchUserData() {
  try {
    const response = await fetch('/api/users');
    return await response.json();
  } catch(err) {
    console.error("사용자 데이터를 불러오는 데 실패했습니다.");
  }
}
```

주석으로 **"왜" 이 코드가 필요한지** 설명하면 좋습니다.

### 나쁜 주석의 예

```javascript
// 변수 x에 10을 할당합니다
let x = 10;

// i를 1 증가시킵니다
i = i + 1;

// 배열에 요소를 추가합니다
arr.push(item);
```

이런 주석은 코드를 읽으면 이해할 수 있으므로 불필요합니다.

## 주석 활용 팁

### 1. 코드 일부 비활성화하기

테스트할 때 특정 코드를 실행하지 않고 싶을 때:

```javascript
// 옛날 방식 (더 이상 사용하지 않음)
// let oldVariable = 123;

// 새로운 방식을 사용합니다
let newVariable = 456;
```

### 2. TODO 표시

나중에 수정하거나 추가할 부분을 표시합니다:

```javascript
function processUserData() {
  // TODO: 입력 데이터 검증 추가
  // TODO: 에러 핸들링 개선

  console.log("사용자 데이터 처리 중...");
}
```

### 3. 섹션 구분

큰 코드 블록을 섹션으로 나눕니다:

```javascript
// ============ 초기화 ============
let users = [];
let currentUser = null;

// ============ 유틸리티 함수 ============
function addUser(name) {
  // ...
}

// ============ 이벤트 핸들러 ============
button.addEventListener("click", function() {
  // ...
});
```

## 주석 작성 시 주의사항

- **너무 많은 주석은 피하세요**: 코드를 읽으면 이해할 수 있는 부분까지 주석 처리하면 오히려 가독성 악화
- **코드와 주석이 일치하도록 하세요**: 코드를 수정했으면 주석도 함께 수정
- **예민한 정보는 주석에 넣지 마세요**: 비밀번호나 API 키 같은 정보는 절대 주석에 넣지 말 것
- **명확하게 작성하세요**: 주석은 간단하고 이해하기 쉽게

## 주석의 중요성

좋은 주석은:
- 코드의 의도를 명확하게 함
- 버그 수정 시간을 단축
- 팀원과의 소통을 원활하게 함
- 나중에 자신의 코드를 다시 봤을 때 이해하기 쉽게 함

**"좋은 코드는 자명해야 하지만, 좋은 주석은 그 코드가 왜 필요한지 설명합니다."**
